Cop Quest

  • High Concept: Broken
  • Player Experience Goals: I was hoping to make this into an adventure game.
  • Genre: Adventure game.
  • Game play: You can aimlessly walk around and you can crash the game if you walk into the door that has incomplete functionality.
  • Prototype goal: I wanted to build upon my knowledge base to give me more powerful tools to make games down the line. I learned how to make and use terrains in tiled, as well as how to create rules for automapping, and creating objects in Tiled and manipulating them in Phaser. My collision layer is almost entirely automapped, and the player's starting point is a part of the Object layer in Tiled. The project stalled at the door object, however, as the action I was attempting to ascribe to the door was to change tilemaps to move from the overworld to the interior of a building, but trying to set the layers for the tilemap the second time around just results in undefined layers and crashing. I have not been able to find any examples of multiple tilemaps being used in a traditional manner, but I had read that it's something that may need to be done using game states, which I will have to look into for next time.
  • Features: Now with 25% more Electrolytes.
  • Setting/Premise: It appears to be a grassy world with some water and dirt, and a singular character who walks around.

  • Target Audience: I'd be disappointed if this did have an audience as is.
  • Play time: Minimal.
  • Strategies: Nothing can help you in this desolate world of nothing to do.
